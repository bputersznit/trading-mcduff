Most Important Discovery
------------------------
The major reconstruction problems were NOT primarily caused by side inversion.

What fixed the book substantially:
1. Use only front-month MNQZ5
2. Use exact instrument_id 42001149
3. Filter out absurd prices using min/max price band
4. Preserve raw Databento action semantics:
   - A -> add
   - M -> modify
   - C -> cancel
   - R -> clear
   - T -> trade_aggressor
   - F -> fill_resting
   - N -> none
5. Treat T and F differently:
   - trade_aggressor should NOT mutate the resting book
   - fill_resting SHOULD decrement resting liquidity
6. Process rows in event batches grouped by ts_event_ns
7. Validate the book only after the event batch completes
8. Use flags bitmask correctly:
   - 128 = F_LAST
   - F_LAST means the row is the final record in the event for that instrument

Important Databento Notes
-------------------------
flags is a bitmask:
- 128 = F_LAST
- 64 = F_TOB
- 32 = F_SNAPSHOT
- 16 = F_MBP
- 8 = F_BAD_TS_RECV
- 4 = F_MAYBE_BAD_BOOK
- 2 = F_PUBLISHER_SPECIFIC

action semantics:
- A = add resting liquidity
- M = modify existing resting order
- C = cancel/remove resting liquidity
- R = clear book
- T = aggressor-side trade info
- F = resting-side fill
- N = no-op informational event

Current Remaining Problem
-------------------------
The remaining reconstruction issue is likely modify semantics.

The current relaxed book does not keep per-order state.

Likely next improvement:
- maintain order_id -> (side, price, size)
- on modify:
  - remove old order state from old price level
  - apply new order state at new price level
- on cancel:
  - decrement known order
- on fill_resting:
  - decrement known order
- on trade_aggressor:
  - record only, do not decrement book
